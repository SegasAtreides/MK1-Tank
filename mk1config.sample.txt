# MK1 Tank config - sample file.
#
# This is /mk1config.txt as it lives on the board's own internal flash
# (LittleFS "storage" partition - see main/mkh_storage.h). It is not read
# from a microSD card: an earlier microSD-based design was retired (see
# main/mkh_sdcard.h for why) in favor of internal storage. The on-device
# editor (tap the settings icon on the dashboard) reads and writes this
# file for you; hand-editing it isn't part of the normal workflow, but
# this file documents the format for anyone who wants to. The firmware
# reads it once at boot only - there is no hot-reload, so an edit requires
# a reboot to take effect.
#
# If the partition is absent, the file is missing, or a line is
# malformed, the firmware logs the reason and falls back to its
# compiled-in defaults for the affected port(s) - it never fails to boot
# over a config problem.
#
# Format: one line per BINDING (WO13: a port can carry more than one -
# see below). Repeated lines for the same port ACCUMULATE rather than
# replace, up to 8 bindings per port.
#   HUB<n>_PORT_<letter> = <input> [invert=yes|no] [max=0-100] [curve=linear] [mode=proportional|momentary|latched]
#
# HUB<n>: app-facing hub number, 1-3, matching the hub's own blue-flash
#         LED blink count when you set its address (HUB1/2/3, not the
#         internal protocol device slots 0/1/2).
# PORT_<letter>: physical output port on the hub, A-F, matching the
#         letters silkscreened on the hub.
# <input>: which controller input drives this port. Full token list
#         (see main/mkh_config.c's kInputTokens, and README.md):
#         LSNS, LSEW, RSNS, RSEW - stick axes
#         LT, RT                - triggers
#         A, B, X, Y            - face buttons
#         LB, RB                - bumpers
#         DUP, DDN, DLT, DRT    - D-pad
#         L3, R3                - stick clicks
# invert: "yes" flips the input's sign before mapping (default: no).
# max:    caps the output range as a percentage of full scale, applied
#         symmetrically around neutral (default: 100).
# curve:  reshapes the input->output response. v1 only implements
#         "linear" (the default) - any other value is accepted but
#         logged as unsupported and treated as linear.
# mode:   only meaningful for non-axis inputs (ignored for LSNS/LSEW/
#         RSNS/RSEW). "proportional" (triggers only), "momentary"
#         (default for buttons/D-pad/stick-clicks - port follows the
#         press), or "latched" (each press toggles the port and holds
#         it; runtime-only, always boots off).
#
# Lines starting with # are comments (this file's own schema-version line
# is one). Blank lines are ignored. Unknown keys or malformed lines are
# logged and skipped - parsing continues with the rest of the file.
#
# WO13 multi-binding: bindings on the same port are ALTERNATIVES - ways
# to reach that port, used one at a time - not a mixer. If you touch two
# at once (e.g. a stick AND a button both bound to HUB1_PORT_A), their
# outputs sum and clamp to +/-100%; an idle/unpressed binding always
# contributes zero, so in the normal case ("only one touched at a time")
# this is exactly the same as picking whichever one is live.

# Reproduces the compiled-in default mapping:
HUB1_PORT_A = LSNS invert=no max=100 curve=linear
HUB2_PORT_A = RSNS invert=no max=100 curve=linear

# latched toggle port (commented out):
# HUB1_PORT_C = RT max=75 mode=momentary
# HUB1_PORT_D = A mode=latched
